# 계약서 시각화 레퍼런스

## 1. 대시보드 디자인 원칙

### 핵심 원칙
- **사용자 중심 설계**: 임대인/임차인/중개인 각 사용자 니즈 파악
- **시각적 계층**: 중요 정보는 좌측 상단, 덜 중요한 정보는 우측 하단
- **단순성**: 핵심 정보만 선별하여 간결하게 표시
- **일관성**: 레이아웃, 색상, 폰트 일관되게 사용
- **가독성**: 색상 대비, 적절한 폰트 선택

---

## 2. 전세 계약 대시보드 핵심 요소

### A. 계약 현황 요약
- 활성 계약 수
- 만료 예정 계약 수
- 미납/연체 현황
- 계약 단계별 현황 (체결 대기 → 진행 중 → 만료 → 종료)

### B. 재정 현황 시각화
- 월별/연도별 임대료 수입/지출
- 전세금 대출 이자, 관리비, 수리비
- 미납액 및 연체율 추이 그래프

### C. 계약 세부 정보
- 개별 계약서 링크 (전자계약 포함)
- 임차인 정보 및 연락처
- 특약사항, 전세자금대출 유무
- 계약 만료 알림 및 재계약 스케줄

### D. 부동산 자산 현황
- 보유 전세 물건 리스트
- 공실 현황 및 기간

---

## 3. 시각화 유형별 활용

### A. 플로우차트 (Flowchart)
- 계약 프로세스 단계 표현
- 의사결정 분기점 시각화
- 법적 절차 및 워크플로우

**활용 예시:**
- 전세 계약 체결 프로세스
- 위험도 평가 의사결정 트리
- 보증보험 가입 절차

### B. 네트워크 그래프 (Network Graph)
- 계약 당사자 간 관계 표현
- 법률/판례 간 연결 관계
- 계약 엔티티 간 연결 (당사자, 위치, 조건)

**활용 예시:**
- 임대인-임차인-중개사 관계도
- 권리관계 네트워크
- 보증기관 연결 관계

### C. 계층형 트리 (Hierarchical Tree)
- 조직 구조 표현
- 법률 조항 계층 구조
- 계약 조건의 분류

**활용 예시:**
- 계약 조항 분류 트리
- 리스크 카테고리 분류
- 체크리스트 구조화

### D. 타임라인 (Timeline)
- 계약 일정 시각화
- 주요 이벤트 순서 표현
- 만료/갱신 일정

**활용 예시:**
- 계약금 → 중도금 → 잔금 일정
- 전입신고/확정일자 타임라인
- 계약 만료 및 재계약 일정

## 5. 계약 대시보드 핵심 KPI

| 지표 | 설명 |
|------|------|
| 총 활성 계약 수 | 현재 진행 중인 계약 건수 |
| 갱신 예정 계약 | 만료 임박 계약 (30/60/90일) |
| 계약 가치 | 총 계약금액 및 월 수익 |
| 결제 준수율 | 정시 결제 비율 |
| 리스크 스코어 | 계약별 위험도 점수 |
| 규정 준수율 | 법적 요건 충족 비율 |

## 6. 차트 유형 선택 가이드

| 데이터 유형 | 권장 차트 |
|-------------|-----------|
| 비교 | 막대 차트, 그룹화된 막대 |
| 추세/시간 | 라인 차트, 영역 차트 |
| 구성/비율 | 파이 차트, 도넛 차트 |
| 분포 | 히스토그램, 박스 플롯 |
| 관계 | 네트워크 그래프, 산점도 |
| 계층 | 트리맵, 선버스트 |
| 지리 | 지도 시각화 |
