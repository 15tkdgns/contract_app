# 계약서 시각화 사례 레퍼런스 & 전세계약 서비스 적용 가이드

## 📚 개요

이 문서는 **법률/계약 시각화의 실제 사례**, **시각화 기법**, 그리고 **전세 사기 방지용 임대차계약 시각화 서비스**에 적용할 수 있는 패턴을 정리한 자료입니다.

## 1. 주요 논문 & 연구 사례

### 1-1. CartaFirm O&M 아웃소싱 계약 시각화 사례 (2016) ⭐ 가장 중요

**논문**: *Exploring contract visualization: Clarification and framing strategies to shape collaborative business relationships*  
**저자**: Stefania Passera, Anssi Smedlund, Minna Liinasuo  
**출판**: Journal of Strategic Contracting and Negotiation, 2(1-2), 69-100 (2016)  
**DOI**: https://doi.org/10.1177/2055563616669739

#### 핵심 내용
- **대상**: 유럽 종이/펄프 산업 대형 기업(CartaFirm, €2.9B)의 O&M 아웃소싱 계약
- **목표**: 장기간(최대 2년) 협상 단축, 명확성 향상, 신뢰 구축
- **연구 방법**: 단일 사례 연구 (인터뷰 + 설계 관찰 + 시각화 분석)
- **결과**: 시각화가 **3가지 지식 격차(Knowledge Gap)** 해소에 효과적임을 발견

#### 3가지 Knowledge Gap

| 격차 유형 | 설명 | 전세계약에 맞춘 해석 |
|----------|------|-----------------|
| **Inter-firm** | 판매자–고객 간 서비스/필요 이해 격차 | 임대인 vs 임차인의 권리/의무 이해 부족 |
| **Cross-professional** | 법무, 재무, 영업, 기술 등 부서간 소통 차이 | 법무사, 부동산 중개사, 임차인의 관점 차이 |
| **Temporal** | 계약 단계 간 협조 어려움(협상→계약→시행) | 계약 체결→전입→확정일자→보증금 반환 각 단계 |

#### CartaFirm이 사용한 시각화 기법

| 계약 주제 | 시각화 유형 | 목표 | 전세 적용 예시 |
|----------|-----------|------|-------------|
| **서비스 범위** | 서비스 범위 맵 (아이콘 라이브러리) | 포함/제외 항목 명확화 | 임대 범위 맵: 냉난방 포함? 세탁기? 반려동물? |
| **역할·책임** | 수영로차트(Swimlane Chart) | 누가 뭘 하는지 명확화 | 전입신고, 확정일자, 보험 가입: 누가 책임? |
| **서비스 특성** | 사진/이미지 | 차별화·신뢰 구축 | 집 사진, 근처 편의시설 사진 |
| **가격·수수료** | X/Y축 다이어그램 + 표 | 가격 정당화, 고객 이익 강조 | 전세가율 게이지, 매매가 vs 전세보증금 |
| **KPI·성과** | 표 | 성과 기준 명확화 | 주택 상태(누수, 균열), 중개사 신뢰도 |
| **비용/결제** | 타임라인 + 표 | 납기 명확화 | 중도금 납기, 잔금 납기, 보증금 반환 예정일 |
| **계약 종료** | 플로우차트 | 중도 해지 조건 | 계약 중도해지 조건, 보증금 분쟁 해결 절차 |

#### 시각화의 두 가지 핵심 전략

1. **명확화(Clarification)**
   - 모호한 조항 명확히 하기
   - 중요 정보 두드러지게 하기
   - 한눈에 이해 가능하게 만들기(인지 부하 줄임)

2. **프레이밍(Framing)**
   - **예방적 프레이밍**: 위험, 손실, 감시 중심 → 냉담한 관계
   - **촉진적 프레이밍**: 기회, 이득, 협력 중심 → 신뢰 있는 관계
   - 계약을 "법적 위협" 대신 "협력 도구"로 인식하게 만들기

#### 핵심 통찰: "Visual Template" 개념
- 시각화는 **수정 가능한 형식**으로 슬라이드와 계약 초안에서 유연하게 사용
- 협상이 진행되면서 반복 개선
- 최종 서명 문서에는 "합의 기반의 스냅샷"으로 고정
- 시행팀에 인계할 때는 "프로젝트 매니저 번역본"으로 변환

### 1-2. AS4000 건설계약 시각화 사례 (2019)

**저자**: Damian Curran (건축 계약 전문 변호사)  
**제목**: "An exercise in contract visualisation using AS4000"  
**URL**: https://damiancurran.com/2019/02/21/an-exercise-in-contract-visualisation-using-as4000/

#### 핵심 내용
- **대상**: 호주에서 광범위하게 사용되는 AS4000 표준 건설계약
- **목표**: 비법률가(프로젝트 매니저, 시공자, 현장 감독 등)도 이해할 수 있게 만들기
- **결론**: 시각화는 "본문을 **대체**하지 못하지만 **보완**할 수 있다"

#### 설계 의사결정과정 (Lessons Learnt)

1. **세부 설계 결정의 중요성**
   - 선의 방향, 점선 vs 실선, 색상 일관성 등 작은 결정이 정확성에 큰 영향
   - 본문의 작은 단어 변화도 시각화에 반영되어야 함

2. **시각화는 단순화된 "이상적 시나리오"**
   - 모든 세부사항을 담으면 혼란만 증가
   - 예: "정상 절차"는 표시하지만, 모든 non-compliance 결과는 제외

3. **타임라인 시각화의 한계와 적용 가능 항목**
   - ✅ **적용 가능**: 시간 순서, 문서/이벤트 이름, 당사자 책임
   - ❌ **부적합**: 세부 조건, 모든 위반 결과 (혼란 초래)

4. **비법률가에게 가치 있음**
   - 법적 조언을 대체하지 못하지만, 계약을 전혀 읽지 않을 사람들이 기본 흐름을 이해하게 해줌

#### 구체적 시각화 예시

**Visual 1: 청구 알림 절차**
```
상부 선: 문서/이벤트 (색상: 당사자별 다름)
하부 선: 타이밍 표시 (< 기호로 "xx일 이내" 의미)
CERTIFIED 스탬프: "승인됨" 시각적 표현
```

**Visual 2: 청구 및 인증서 절차**
```
초록색 막대: 결제 표시
일관된 색상 규칙 유지 (사용자가 매번 다시 배우지 않도록)
```

**Visual 3&4: 준공 및 보증금 감액**
```
파란색 막대: 보증금 해제
초록 체크/빨강 X: 승인/거절 직관적 표시
```

## 2. 시각화 기법별 분류 (전세 서비스 적용 매핑)

### 2-1. 타임라인 시각화

**용도**: 시간 순서 프로세스, 기한, 마일스톤

**전세 적용 예시**:
```
계약 체결일 → 중도금 납기 → 잔금 납기 → 전입신고 → 확정일자 → 보증금 반환 예정일
    |            |           |         |         |          |
    ✓           (금액)      (금액)    (기한)    (기한)      (반환액)
```

**설계 팁**:
- 각 단계 위에 담당자(임대인/임차인) 표시
- 타이밍 범위 표시(예: "5일 이내" → "<5")
- 완료된 항목 vs 진행 중 구분(색상/아이콘)
- 중요 기한은 색상으로 강조

### 2-2. 플로우차트 / 의사결정 트리

**용도**: 분기점, 조건부 프로세스, Yes/No 흐름

**전세 적용 예시**:
```
[계약서 입력]
    ↓
[등기부 소유자 = 계약서 임대인?]
    ├─ YES → [권리관계 확인으로 진행]
    └─ NO  → [⚠️ 위험 신호: 명의신탁 의심]
              [법무사 상담 권장]
    ↓
[전세가율 < 70%?]
    ├─ YES → [안전 (녹색)]
    └─ NO  → [높은 위험 (빨강)]
    ↓
[전세 사기 위험도 스코어 계산]
```

**설계 팁**:
- React Decision Tree Flow 또는 React Flow 라이브러리 사용
- 각 노드에 질문 또는 결과 명확히 표시
- 색상으로 위험도 시각화 (녹색/노랑/빨강)

### 2-3. 네트워크 그래프 / 관계도

**용도**: 엔티티(사람, 회사, 부동산) 간 관계 표현

**전세 적용 예시**:
```
          [집(주소)]
           /  |  \
          /   |   \
    [소유자]  [임대인]  [중개인]
        |       |        |
        |    [임차인]  [보증기관]
        |       |        |
    [근저당금액] [보증금] [보증보험]
```

**설계 팁**:
- React Flow의 노드로 각 엔티티 표현
- 간선(edge)으로 관계 표시 (예: "소유–근저당", "임대–보증금")
- 위험 엔티티는 빨강, 안전 요소는 초록 아이콘
- 마우스 호버로 상세 정보 표시

### 2-4. 게이지 / 프로그레스 바 (위험도 지표)

**용도**: 한 가지 수치적 지표 시각화

**전세 적용 예시**:
```
전세가율:
━━━━━━ 60% (안전) ━━━━━━ 90% (위험) ━━━━━━ 120% (매우 위험)
                   현재값: 85% 🔴 높음

보증금 회수 가능성:
━━━━━━ 100% 회수 가능 ━━━━━━ 50% 손실 위험 ━━━━━━ 0% 손실
                현재값: 65% 🟡 중간 위험
```

**설계 팁**:
- Recharts, Nivo 등 React 차트 라이브러리 사용
- 색상 구간 명확히 (녹색 < 노랑 < 빨강)
- 수치 + 텍스트 해석 함께 표시

### 2-5. 매트릭스 / 히트맵 (권리관계)

**용도**: 2차원 데이터 비교 (당사자 × 의무)

**전세 적용 예시**:
```
                | 임대인 책임 | 임차인 책임 | 공동책임 | 보증기관
────────────────┼───────────┼──────────┼──────────┼────────
전입신고        |    ✓      |    ✗     |    -     |   -
확정일자        |    ✗      |    ✓     |    -     |   -
보증보험        |    -      |    -     |    ✓     |   ✓
보증금 반환     |    ✓      |    ✗     |    -     |   보증
월세 납기       |    ✗      |    ✓     |    -     |   -
```

**설계 팁**:
- 색상으로 책임 명확화 (초록=책임, 빨강=책임 없음, 노랑=공동)
- 행과 열을 명확하게 레이블링
- 마우스 호버로 세부사항 표시

### 2-6. 카드형 요약 (핵심 정보)

**용도**: 페이지 상단에 1페이지 요약

**전세 적용 예시**:
```
┌────────────────────────────────┐
│ 전세 계약 한눈에 보기           │
├────────────────────────────────┤
│ 주소: _______________          │
│ 면적: ____ m²                  │
│ 보증금: ____억 ____ 만 원       │
│ 월세: ____만 원                │
│ 계약기간: ____ ~ ____          │
│                               │
│ 📊 위험도: 🔴 높음 (72/100)    │
│ 🎯 주요 위험:                 │
│    • 전세가율 90% (높음)       │
│    • 선순위 근저당 3억         │
│    • 보증보험 미가입           │
└────────────────────────────────┘
```

**설계 팁**:
- 정보 계층화 (중요도순 정렬)
- 아이콘으로 카테고리 구분
- 모바일 친화적 레이아웃

## 3. 전세 사기 예방 서비스에 직접 적용할 패턴

### 3-1. "전세 안전 체크리스트" 플로우

```
[임대차계약 PDF/이미지 업로드]
            ↓
[문서 파싱 → 필드 추출]
(주소, 임대인, 보증금, 월세, 계약기간 등)
            ↓
[외부 데이터 연동]
(등기부 조회, 시세 조회, 전세가율 계산)
            ↓
[위험 신호 자동 탐지]
  ├─ 등기부 소유자 ≠ 계약서 임대인?
  ├─ 전세가율 > 85%?
  ├─ 선순위 근저당 > 보증금?
  ├─ 보증보험 여부?
  ├─ 대항력·확정일자 예정?
  └─ 조항 내 위험 문구?
            ↓
[위험도 스코어 계산 및 표시]
(0~100점, 색상 코딩)
            ↓
[시각화 대시보드 생성]
  ├─ 타임라인: 체결→중도금→잔금→전입→확정→보증금반환
  ├─ 네트워크: 집·임대인·임차인·근저당·보증기관 관계
  ├─ 게이지: 전세가율, 회수 가능성
  ├─ 매트릭스: 권리순위, 당사자 책임
  └─ 카드: 위험도 + 권장사항
            ↓
[AI Q&A 패널]
사용자: "이 집 보증금 돌려받을 수 있어?"
→ 머신: "전세가율 90%, 선순위 근저당 3억원 → 
  경매 시 회수 가능 추정액: 4억원 중 2억~3억원"
```

### 3-2. "권리관계 레이어 맵" (스택 차트)

```
바 차트로 표현:
━━━━━━━━━━━━━━━━━━━━━━━━━━━ 감정가: 10억원
━━━━━━━━━━━━━━━━━━━━ 매매가: 9억원
1층 (선순위근저당): 2억원  🔴 위험
2층 (선순위전세):   1억원  🟡 주의
3층 (내 보증금):    4억원  🟢 안전
미회수 가능: ?              🔴 위험

[경매 시 예상 회수]
내 층(4억)에서 경매가(8억) × (회수율 60%)
= 약 2억원 손실 가능
```

### 3-3. "전세 절차 타임라인" (양쪽 책임 명시)

```
계약 체결       중도금         잔금          전입신고      확정일자      계약 만료
   |            |             |             |            |            |
   ↓            ↓             ↓             ↓            ↓            ↓
[임대인]      [임차인]      [임차인]     [임차인]     [누구?]       [임대인]
보증금       중도금        잔금        신청 후      [선택]        전액반환
수령        지급 예정     지급 예정    5일내 공고   신청·        기한: 7일
              XX일        XX일        목표 기한    확정        이상 연체?
                                                                   ↓ 경매위험
```

## 5. 설계 체크리스트 (CartaFirm + AS4000 경험 토대)

### 시각화 전에 확인할 사항

- [ ] **명확한 목표**: "누가, 무엇을 이해해야 하는가?"
  - 임차인: 보증금 안전, 기한 체크
  - 임대인: 의무사항, 책임 경계
  - 중개사: 완결성, 법적 문제 없음

- [ ] **대상 사용자 맞춤**
  - 비법률가도 이해 가능한가?
  - 모바일/태블릿 친화적인가?
  - 외국인도 이해 가능한가? (아이콘/색상)

- [ ] **일관성 규칙**
  - 색상: 초록(안전), 노랑(주의), 빨강(위험)
  - 아이콘: 체크(확인), X(미확인), ??(불명확)
  - 선: 실선(의무), 점선(선택)
  - 타이밍: < XX (xx일 이내), > XX (xx일 이상)

- [ ] **단순화 원칙**
  - 모든 법적 세부사항을 담지 말 것 (혼란 야기)
  - "이상적 절차"를 표시하되, 모든 예외는 범범 표시
  - 원문 참조 유도

- [ ] **반복 테스트**
  - 실제 전세계약 5건 이상에서 테스트
  - 사용자 피드백 수집 (임차인/중개사/법무사)
  - 프로토타입 개선

### 시각화 후 검증

- [ ] **이해도**: 사용자가 20초 안에 주요 정보 파악?
- [ ] **정확성**: 원문과 시각화 일치?
- [ ] **컴플리트**: 필수 정보 누락?
- [ ] **신뢰**: "이 회사는 진지한가?"를 느껴지는가?

## 6. 전세 사기 사례와 시각화 활용

### Case 1: 깡통전세 (전세가율 과도)

**문제**: 매매가 대비 전세보증금이 과도 → 시세 하락 시 보증금 미회수

**시각화 솔루션**:
```
게이지 + 네트워크:
전세가율 95% 🔴 높음
→ 매매가: 8억
→ 보증금: 7억 6천
→ 경매 시 예상 손실: -2억 이상

권리관계 그래프:
[집(8억)] 
  ├─ [근저당 3억] 🔴
  ├─ [전세권 2억] 🟡
  └─ [내 보증금 7억 6천] 🔴 "위험!"
```

### Case 2: 소유자-임대인 불일치 (명의신탁/가해자)

**문제**: 등기부상 소유자 ≠ 계약서상 임대인 → 명의신탁, 무권한 전세 가능

**시각화 솔루션**:
```
플로우차트:
[등기부 조회: 김철수]
[계약서 임대인: 이영희]
    ↓
    🔴 불일치!
    ↓
[법무사 상담 필수]
→ 실제 소유자와 임대인 확인
→ 위임장 진위 확인
→ 권리관계 재정리
```

### Case 3: 보증보험 미가입

**문제**: 임대인 파산 시 보증금 미회수 가능

**시각화 솔루션**:
```
네트워크 그래프:
[임차인] --보증금--> [임대인]
                      ├─ 보증보험: ❌ 없음 🔴
                      └─ [보증기관 (HUG/SGI)]: 필수!

카드:
🎯 권장사항: 계약서 작성 시 반드시 보증보험 약정 포함
→ 무조건 HUG(주택도시보증공사) 또는 SGI 보증보험 확인
→ 보증 한도 < 보증금이면 부족분 위험
```

### Case 4: 선순위 임차인/근저당 과다

**문제**: 내 보증금보다 선순위 채권이 많으면 경매 시 회수 불가

**시각화 솔루션**:
```
매트릭스/스택차트:
├─ 감정가: 10억원
├─ 1순위 근저당: 4억원 🔴
├─ 2순위 전세: 3억원 🟡
└─ 내 전세: 4억원 🔴 "위험!"

→ 경매가 8억원 기준, 예상 회수: 0원~1억원
```

## 7. 실무 팁: 시작하기

### Phase 1: 프로토타입 (1-2주)

1. **현황 분석**
   - 실제 전세계약 5건 수집
   - 전세 사기 사례 3건 분석
   - 사용자(임차인, 중개사) 인터뷰 (각 30분)

2. **핵심 시각화 3개 선정**
   - 위험도 카드 (게이지)
   - 타임라인 (기한 체크)
   - 권리관계 그래프 (누가 먼저 받는지)

3. **React Flow + Recharts 프로토**
   - 샘플 1건으로 테스트
   - 사용자 피드백 (FGI 또는 1:1)

### Phase 2: MVP (2-4주)

1. **모든 필드 파싱**
   - 계약서 PDF → JSON 추출
   - OCR + NLP로 필드 자동 감지

2. **외부 데이터 연동**
   - 등기부: 대법원 인터넷 등기소 API
   - 시세: 국토교통부 오피스 API 또는 부동산114
   - 계산: 전세가율, 회수 가능액 알고리즘

3. **대시보드 완성**
   - 위험도 점수화
   - 4-5개 시각화 패널
   - AI Q&A (임시로 GPT API 활용)

### Phase 3: 런칭 (4주+)

1. **보안 및 개인정보**
   - 계약서 암호화 저장
   - 감시 로그

2. **사용성 테스트**
   - 베타 사용자 50명 모집
   - 피드백 → 개선 반복

3. **마케팅**
   - 전세 관련 커뮤니티 (당신의 집값, 부동산114 등)
   - 대학생/사회초년생 타겟
